# SynapDesk для Windows — инструкция тестера

Спасибо, что помогаете с альфа-тестом. Ниже — всё необходимое для установки и проверки облака.

## Что скачать

В этой папке должен лежать установщик SynapDesk, например:

- `SynapDesk_0.1.0_x64-setup.exe` (NSIS), или
- `SynapDesk_0.1.0_x64_en-US.msi`

Если файла нет — дождитесь ссылки от организатора теста или скачайте с GitHub Releases.

**Требования:** Windows 10/11, 64-bit, [WebView2](https://developer.microsoft.com/microsoft-edge/webview2/) (обычно уже установлен).

---

## Установка

1. Скачайте `.exe` из этой папки или по ссылке организатора.
2. Запустите установщик.
3. Если Windows SmartScreen пишет «Неизвестный издатель»:
   - нажмите **Подробнее** → **Выполнить в любом случае**  
   (сборка альфа, без коммерческой подписи — это ожидаемо).
4. После установки запустите **SynapDesk** из меню Пуск.

## Облако и сервер (обязательно для теста sync/аккаунта)

Приложение по умолчанию ожидает API по адресу:

```text
http://192.168.0.18:8080
```

Если организатор дал **другой URL** — используйте его.

### Проверка, что сервер доступен

В PowerShell или cmd:

```powershell
curl http://192.168.0.18:8080/health
```

Должен ответить JSON со статусом «ok» / healthy. Если **таймаут или ошибка сети**:

- вы в той же Wi‑Fi/LAN, что и сервер (или подключены к VPN организатора);
- брандмауэр на сервере пропускает порт **8080**;
- уточните актуальный IP у организатора.

### Настройка в приложении

1. Откройте **Настройки** → **Облако**.
2. В поле **URL API** (SYNAPDESK_API_URL) укажите адрес сервера, например:  
   `http://192.168.0.18:8080`  
   Без слэша в конце.
3. Сохраните.

Альтернатива перед первым запуском (для продвинутых):

```powershell
setx SYNAPDESK_API_URL "http://192.168.0.18:8080"
```

Перезапустите SynapDesk после `setx`.

## Регистрация и вход (альфа)

В альфа-сборке почта может не отправляться реально — коды пишутся в лог сервера.

| Шаг | Действие |
|-----|----------|
| Регистрация | Укажите email и пароль аккаунта |
| Подтверждение | Введите код из письма **или** (если сказал организатор) фиксированный тест-код: **`424242`** |
| Вход | Email + пароль |

Если код не приходит — спросите организатора (код в логах сервера при `SYNAPDESK_EMAIL_MODE=log`).

## Что проверить в тесте

- [ ] Приложение запускается, создаётся/открывается локальное хранилище (vault).
- [ ] Заметки сохраняются, поиск работает.
- [ ] **Облако:** регистрация → подтверждение → вход.
- [ ] **Облако:** привязка vault к аккаунту (если есть в UI).
- [ ] **Sync / файлы:** загрузка/скачивание с сервера (если включено в этой версии).
- [ ] Запишите баги: что делали, ожидание, что получилось, скриншот.

**AI (пунктуация, выжимка)** в альфе **не обязателен** — нужны отдельно Ollama и LLM-шлюз на машине разработчика. Для cloud-теста можно не трогать.

## Где лежат ваши данные

Локальные vault и настройки — в профиле пользователя Windows, каталог данных приложения SynapDesk (организатор подскажет путь при необходимости). Пароль vault **не** хранится на сервере в открытом виде — сервер видит только зашифрованные blob’ы.

## Частые проблемы

| Проблема | Что сделать |
|----------|-------------|
| «Не удаётся подключиться к API» | Проверьте URL в Настройках → Облако и `curl .../health` |
| SmartScreen блокирует установку | Подробнее → Выполнить |
| Приложение не стартует | Установите [WebView2 Runtime](https://developer.microsoft.com/microsoft-edge/webview2/) |
| Неверный код email | Уточните у организатора; в альфе часто `424242` |
| Старый URL `127.0.0.1:8080` | Замените на LAN-адрес сервера в настройках |
