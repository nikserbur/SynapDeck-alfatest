# SynapDesk для macOS — инструкция тестера

Спасибо, что помогаете с альфа-тестом. Ниже — всё необходимое для установки и проверки облака.

## Что скачать

В этой папке должен лежать образ диска SynapDesk, например:

- `SynapDesk_0.1.0_aarch64.dmg` (Mac с Apple Silicon M1/M2/M3), или
- `SynapDesk_0.1.0_x64.dmg` (Intel Mac)

Если файла нет — дождитесь ссылки от организатора или скачайте с GitHub Releases.

**Требования:** macOS 11+ (уточните у организатора минимальную версию для вашей сборки).

---

## Установка

1. Откройте `.dmg` двойным щелчком.
2. Перетащите **SynapDesk** в папку **Программы** (Applications).
3. Извлеките диск (извлечь образ) — необязательно, но удобно.
4. Запустите SynapDesk из Программ.

### Если macOS пишет «разработчик не может быть проверен»

Сборка альфа, без Apple notarization — это нормально для внутреннего теста.

**Способ 1 (рекомендуется):**  
ПКМ по приложению → **Открыть** → **Открыть** в диалоге.

**Способ 2:**  
Системные настройки → **Конфиденциальность и безопасность** → разрешить запуск SynapDesk.

**Способ 3 (только если организатор попросил):** в Терминале:

```bash
xattr -cr /Applications/SynapDesk.app
```

## Облако и сервер (обязательно для теста sync/аккаунта)

Приложение по умолчанию ожидает API по адресу:

```text
http://192.168.0.18:8080
```

Если организатор дал **другой URL** (домен, другой IP) — используйте его.

### Проверка, что сервер доступен

В Терминале:

```bash
curl -s http://192.168.0.18:8080/health
```

Должен ответить JSON со статусом «ok» / healthy. Если **таймаут**:

- вы в той же Wi‑Fi/LAN, что Windows-сервер с API, или в VPN организатора;
- с Mac недоступен IP сервера (уточните IP);
- на сервере открыт порт **8080**.

### Настройка в приложении

1. **Настройки** → **Облако**.
2. Поле **URL API** — например `http://192.168.0.18:8080` (без `/` в конце).
3. Сохраните.

Перед первым запуском (опционально):

```bash
export SYNAPDESK_API_URL="http://192.168.0.18:8080"
open -a SynapDesk
```

Для постоянной переменной добавьте строку в `~/.zshrc` или `~/.bash_profile`.

## Регистрация и вход (альфа)

| Шаг | Действие |
|-----|----------|
| Регистрация | Email + пароль аккаунта |
| Подтверждение | Код из письма **или** тест-код от организатора: **`424242`** |
| Вход | Email + пароль |

Коды в альфе часто только в логах сервера — не ждите реальное письмо без настройки SMTP.

## Что проверить в тесте

- [ ] Запуск, создание/открытие vault.
- [ ] Заметки, поиск, базовый редактор.
- [ ] **Облако:** регистрация → подтверждение → вход.
- [ ] Привязка vault к аккаунту (если есть в UI).
- [ ] Sync / резервные файлы на сервере (если в этой версии).
- [ ] Баги: шаги, ожидание, факт, скриншот.

**Локальный AI** (Ollama, выжимка текста) для cloud-теста **не нужен**.

## Где лежат данные

Vault и `settings.json` — в каталоге данных SynapDesk в вашем домашнем каталоге (организатор подскажет путь). Содержимое заметок на сервере хранится **зашифрованным**; пароль vault сервер не знает.

## Частые проблемы

| Проблема | Что сделать |
|----------|-------------|
| Не открывается приложение | ПКМ → Открыть; или `xattr -cr` (см. выше) |
| API недоступен | Проверьте Wi‑Fi/VPN и URL в Настройках → Облако |
| `127.0.0.1:8080` не работает | Нужен **LAN-IP сервера**, не localhost с Mac |
| Неверный код | Спросите организатора; часто `424242` в dev |
| Rosetta | Для Intel-сборки на Apple Silicon — установите Rosetta, если попросит система |
